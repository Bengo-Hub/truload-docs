# Operations

The operations guide documents how TruLoad is built, deployed, secured,
and run. It is the reference for platform engineers, SREs, and integrators.

## Chapters

[Architecture](architecture.md)
: System topology, data stores, service boundaries, transaction path.

[Deployment](deployment-and-environments.md)
: Where TruLoad runs (cluster, hostnames, resources), how test and
  production are segregated, deploy and rollback procedures.

[Security](security-assurance.md)
: Implemented controls, audit findings and remediation, verification
  checklist.

[Integrations (M-PESA)](integrations-mpesa.md)
: Pesaflow / eCitizen flow, callbacks, reconciliation, idempotency.

[Backup, DR & troubleshooting](backup-dr-troubleshooting.md)
: Backup schedule and retention, restore drill, disaster-recovery
  sequence, closed-space operation, symptom-to-cause map.
